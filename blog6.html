End-to-End API Automation Testing with Rest Assured

API automation testing is crucial for validating the functionality of backend services. Rest Assured is a powerful Java-based library that simplifies testing RESTful APIs.

Common HTTP Status Codes

  • 200 OK - The request was successful.
  • 201 Created - A new resource was successfully created.
  • 400 Bad Request - The request is invalid or malformed.
  • 401 Unauthorized - Authentication is required.
  • 403 Forbidden - The server understood the request but refuses to authorize it.
  • 404 Not Found - The requested resource could not be found.
  • 500 Internal Server Error - A generic error message for server failures.

Real-Time API Testing Process

  1. Define test cases based on API requirements.
  2. Set up Rest Assured dependencies in the project (Maven/Gradle).
  3. Write API test scripts for different request methods (GET, POST, PUT, DELETE).
  4. Validate response status codes, headers, and response body.
  5. Integrate tests into TestNG or JUnit for structured execution.
  6. Execute tests in local, staging, or production environments.
  7. Automate API execution in CI/CD pipelines (Jenkins, GitHub Actions).

Example: API Test with Rest Assured

            
import io.restassured.RestAssured;
import io.restassured.response.Response;
import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;
import org.testng.annotations.Test;

public class APITest {
    @Test
    public void testGetUser() {
        RestAssured.baseURI = "https://jsonplaceholder.typicode.com";
        
        given()
            .when()
            .get("/users/1")
            .then()
            .statusCode(200)
            .body("name", equalTo("Leanne Graham"));
    }
}

Execution Process

  • Run tests using mvn test or gradle test.
  • Generate reports using TestNG, Extent Reports, or Allure.
  • Execute in a CI/CD pipeline using Jenkins/GitHub Actions.
  • Store test results in logging tools like ELK or Grafana.
